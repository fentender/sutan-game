# 苏丹的游戏 - Mod 合并管理器 使用手册

> 版本：v1.2.0 | 更新日期：2026-04-19

---

## 一、简介

「苏丹的游戏」支持创意工坊 Mod，但当多个 Mod 修改了同一个游戏文件时，它们无法同时启用——后启用的 Mod 会完全覆盖前面的 Mod。

**Mod 合并管理器**就是为了解决这个问题而设计的工具。它以游戏本体文件为基础，按你设定的优先级逐层合并各个 Mod 的修改，最终生成一个**合成 Mod**。启用这个合成 Mod 即可同时享受所有 Mod 的内容。

---

## 二、安装与首次启动

### 安装方式

**方式一：直接运行（推荐）**

从发布页面下载 `.exe` 文件，双击即可运行，无需安装 Python。

**方式二：Python 运行**

需要 Python 3.10 以上版本。

```
pip install -r requirements.txt
python -m src.main
```

或双击项目目录下的 `run.bat`。

### 首次启动

首次启动时会弹出路径设置窗口，需要填写：

- **游戏安装目录**：例如 `D:\SteamLibrary\steamapps\common\Sultan's Game`
- **创意工坊目录**：通常可以从游戏路径自动推导，无需手动填写

填写游戏路径后，Workshop 路径会自动推导填充。确认路径正确后点击「确定」。

首次运行还会自动生成 Schema 规则文件，此时会显示一个进度条，耐心等待即可（仅首次需要，之后不再重复）。

## 三、主界面介绍

主界面分为以下几个区域：

| 区域 | 位置 | 功能 |
|------|------|------|
| **Mod 列表** | 左侧 | 显示所有已安装的 Mod，管理启用状态和优先级 |
| **Mod 详情** | 右上 | 显示当前选中 Mod 的预览图、名称、版本、标签和描述 |
| **操作按钮** | 中部 | 执行合并、清理合成 Mod、选择合并模式等 |
| **覆盖详情** | 下方 | 展示各 Mod 对游戏文件的修改情况和冲突 |
| **日志面板** | 底部 | 显示操作过程中的信息、警告和错误 |

## 四、Mod 管理

### 启用 / 禁用 Mod

每个 Mod 项前面有一个**复选框**，勾选即启用该 Mod 参与合并，取消勾选则排除。

列表底部提供「全选」和「全不选」按钮，方便批量操作。

### 调整优先级

**优先级规则：列表中越靠下的 Mod 优先级越高。** 当多个 Mod 修改了同一个字段时，优先级最高的 Mod 的修改会被最终采用。

调整优先级有两种方式：

- **拖拽排序**：直接拖动 Mod 项到目标位置
- **上下按钮**：点击 Mod 项右侧的 ▲ / ▼ 按钮

### 查看 Mod 详情

点击某个 Mod 后，右侧详情面板会显示：

- 封面预览图
- Mod 名称和版本号
- 标签列表
- 完整描述文本

## 五、版本风险提示

工具会自动检测每个 Mod 的更新时间，并与游戏本体版本进行比对。在 Mod 列表中，每个 Mod 名称前面可能出现以下图标：

| 图标 | 含义 | 风险程度 |
|------|------|----------|
| 🔵 蓝色感叹号 | Mod 版本落后于游戏更新，但该 Mod 是**纯增量**的（只添加新内容，不修改本体文件） | 低风险 |
| 🟡 黄色三角 | Mod 的最后更新时间早于游戏的小版本更新 | 较低风险 |
| 🔴 红色感叹号 | Mod 的最后更新时间早于游戏的大版本更新（2026年3月31日） | 较高风险 |

**应对建议：**

- 蓝色图标的 Mod 通常可以放心使用
- 黄色图标的 Mod 建议使用「智能合并」模式
- 红色图标的 Mod 需要特别注意，建议合并后检查覆盖详情面板中是否存在冲突，必要时手动校对

## 六、合并模式

### 设置方式

- **全局模式**：在操作按钮区域右侧的下拉菜单中选择，对所有 Mod 生效
- **单 Mod 模式**：在每个 Mod 项上的下拉菜单中单独设置，默认为「跟随全局」

单 Mod 的设置会覆盖全局设置。

### 三种模式详解

#### 智能合并（推荐）

- 正常应用 Mod 的新增和修改操作
- **选择性处理删除操作**：禁止旧 Mod 删除游戏新版本新增的数组元素，避免丢失游戏更新内容
- 适合大多数场景，尤其是混合使用新旧 Mod 时

#### 正常合并

- 严格按照 Mod 的修改内容执行，包括新增、修改和**所有删除**
- 如果 Mod 版本过时，可能会删掉游戏新版本新增的内容
- 适合 Mod 与游戏版本完全匹配的情况

#### 直接替换

- 不做字段级合并，直接用 Mod 的文件整个替换游戏本体文件
- 如果多个 Mod 修改了同一文件，只保留优先级最高的那个 Mod 的版本
- 适合某个 Mod 的功能**必须完整实现**、不能被其他 Mod 干扰的场景。此时建议将该 Mod 放在最高优先级（列表最下方）

### 模式选择建议

| 场景 | 推荐模式 |
|------|----------|
| 日常使用，多 Mod 混合 | 智能合并 |
| 所有 Mod 都与游戏版本匹配 | 正常合并 |
| 某个 Mod 的效果必须完整保留 | 对该 Mod 单独设为直接替换 |

## 七、冲突检测与覆盖详情

### 覆盖详情面板

当你启用 Mod 并调整完顺序后，工具会自动分析所有 Mod 对游戏文件的修改情况，结果展示在覆盖详情面板中。

面板以**树形结构**组织：

- **第一级**：被修改的文件（显示文件路径）
- **第二级**：该文件中被修改的具体字段

每个条目显示三列信息：

| 列 | 说明 |
|----|------|
| 文件/字段 | 文件相对路径，或具体的字段路径 |
| 覆盖链 | 例如 `[本体] ← ModA ← ModB`，表示该字段依次被 ModA 和 ModB 修改，最终值取 ModB |
| 最终值 | 合并后的最终结果 |

### 筛选和搜索

面板顶部提供四种**筛选标签**：

| 标签 | 说明 |
|------|------|
| **所有** | 显示全部修改 |
| **普通** | 仅显示无冲突的修改 |
| **数组合并** | 仅显示涉及数组合并的文件（需要额外关注） |
| **冲突** | 仅显示存在冲突的文件（多个 Mod 修改同一字段且值不同） |

还提供一个**搜索框**，可以按文件名或 Mod 名进行实时搜索。

### 颜色标记

文件级节点会用颜色标记其状态：

- **黑色**：普通修改，无冲突
- **橙色 + "(冲突!)"**：该文件中存在字段冲突
- **黄色 + "(数组合并)"**：该文件涉及数组元素的合并
- **橙色 + "(冲突, 数组合并)"**：同时存在冲突和数组合并

字段级子节点中，新增的条目会用**绿色**标记「新增」。

## 八、Diff 对比与手动编辑

### 打开 Diff 窗口

在覆盖详情面板中，**双击**某个文件节点即可打开 Diff 对比窗口。

### 窗口布局

- **左侧编辑器**（只读）：显示合并前的文件内容
- **右侧编辑器**（可编辑）：显示合并后的结果
- **顶部 Tab 标签**：每个修改该文件的 Mod 各有一个 Tab，可以切换查看不同 Mod 的修改

Tab 标签的颜色也有含义：

- 黑色 = 无冲突
- 橙色 = 存在冲突
- 蓝色 = 存在数组合并

### 颜色含义

Diff 视图中，不同行用不同背景色标记：

| 颜色 | 含义 |
|------|------|
| 绿色 | 新增的行 |
| 红色 | 被删除的行 |
| 黄色 | 被修改的行 |
| 橙色 | 冲突的行（多个 Mod 修改了同一字段） |
| 蓝色 | 用户手动覆写的行 |
| 灰色 | 对齐填充行（无实际内容） |

### 导航

窗口中提供「▲ 上一个变化」和「▼ 下一个变化」按钮，可以快速跳转到各个差异位置。中间会显示当前变化块的序号。

支持 **Ctrl+F** 打开搜索框进行文本搜索。

### 手动编辑

如果自动合并的结果不满意，你可以直接在右侧编辑器中修改内容，然后点击「保存」按钮保存覆写。**保存后，被覆写的行会以蓝色高亮显示**，表示该内容是你手动指定的，后续合并不会覆盖它。

点击「重置为默认」可以撤销覆写，恢复为自动合并的结果。

点击「格式化」可以自动调整 JSON 缩进格式。

## 九、删减报告

点击操作区域的「查看删减报告」按钮，可以查看各 Mod 删除了哪些游戏本体的字段。

### 报告结构

- **"所有" Tab**：按文件组织，显示所有 Mod 的删减汇总
- **单 Mod Tab**：每个 Mod 一个 Tab，仅显示该 Mod 的删减
- **"统计" Tab**：按容器聚合的统计视图

### 查看详情

**双击**任何删减条目可以打开预览窗口，查看完整的文件内容。被删除的行会用红色标记，当前选中的删减用深红色标记。

---

## 十、执行合并与部署

### 操作步骤

1. 确认已启用需要合并的 Mod 并调整好优先级
2. 检查覆盖详情面板中是否存在需要处理的冲突
3. 点击「执行合并」按钮
4. 在弹出的对话框中输入合成 Mod 的文件夹名称（默认为 `0000000001`，仅限英文、数字和下划线）
5. 等待合并完成（会显示进度条）
6. 合并成功后，合成 Mod 会自动部署到 Workshop 目录

### 合并完成后

合并成功后会弹出提示，显示合并的文件数和输出目录。此时你可以：

- 在游戏中启用合成 Mod，**同时禁用**其他已被合并的 Mod
- 启动游戏验证效果

---

## 十一、清理合成 Mod

点击「清理合成 Mod」按钮，工具会列出所有之前生成的合成 Mod。勾选要删除的项目后确认即可清理。

---

## 十二、JSON 解析错误修复

部分 Mod 的 JSON 文件可能存在格式问题（如语法错误）。当工具遇到无法解析的文件时，会弹出修复窗口。

窗口中会：

- 显示具体的错误位置和错误信息
- 提供可编辑的文本区域，你可以手动修复语法错误
- 错误所在行会用红色高亮标记

修复后点击「保存」验证格式是否正确。如果不想修复，可以点击「忽视剩余，默认处理」跳过。未修复的文件将无法参与合并。

- 点击「清理」按钮可以清空日志列表

## 十四、其他功能

### 路径设置

通过菜单栏「文件」菜单可以随时修改：

- 游戏安装目录
- 创意工坊目录
- 本地 Mod 目录（用于存放非 Workshop 来源的 Mod）

## 十五、常见问题

### Mod 优先级怎么理解？

列表中越靠下的 Mod 优先级越高。当多个 Mod 修改了同一个字段时，最终结果取优先级最高（最靠下）的 Mod 的值。因此，你最看重的 Mod 应该放在列表最下方。

### 合并后游戏内容缺失怎么办？

可能是旧版本 Mod 在「正常合并」模式下删除了游戏新版本的内容。尝试以下方法：

1. 将全局合并模式切换为「智能合并」
2. 检查覆盖详情面板，查看是否有被删除的字段
3. 查看删减报告，定位是哪个 Mod 导致的删减
4. 对问题 Mod 尝试调整优先级或更换合并模式

### 什么时候该用直接替换？

当你确定某个 Mod 的功能必须完整实现，不希望被字段级合并影响时，可以对该 Mod 单独设为「直接替换」。注意：此模式下，如果该 Mod 修改的文件也被其他 Mod 修改了，其他 Mod 的修改会被完全覆盖。

### 冲突标记一定代表有问题吗？

不一定。冲突只表示多个 Mod 修改了同一个字段且值不同，最终值取优先级最高的 Mod。如果你本来就希望高优先级 Mod 的修改覆盖低优先级的，那这个冲突是正常的。但如果你希望两个 Mod 的修改都生效，就需要手动编辑合并结果。

### 合成 Mod 在哪里？

合成 Mod 会自动部署到你设置的创意工坊目录下，文件夹名称就是你在合并时输入的名称（默认为 `0000000001`）。在游戏中启用该合成 Mod 即可。

### 使用合成 Mod 后需要禁用其他 Mod 吗？

**需要。** 合成 Mod 已经包含了所有被合并的 Mod 的修改内容。如果同时启用合成 Mod 和原始 Mod，可能会导致重复或冲突。建议在游戏中仅启用合成 Mod，禁用其他已被合并的 Mod。
