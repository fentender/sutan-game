# 条件
### 条件-通用
说明：仪式，幕后，结局，后日谈激活对应结果动作的通用条件格式。
##### table_have——当前游戏存在闲置的对应卡牌(或不存在)
- 指令格式："table_have.筛选器":1（"!table_have.筛选器":1）
- 闲置的：在手牌中或者没有在仪式中的
- 如果要表示小于，等于，小于等于，大于等于，大于，可添加对应的符号
    - 如果不配置符号，默认是大于等于
- 穷举筛选器配置格式
```json
"table_have.卡牌id":1, 
"table_have.卡牌id.卡牌tag":1,
"table_have.卡牌tag":1,
"table_have.卡牌tag.卡牌tag":1,
"table_have.卡牌类型":1,
"table_have.卡牌类型.卡牌tag":1,
-----------范例如下-------------------
"table_have.2000006":1,//场上至少存在1张闲置的ID为2000006的卡牌
"table_have.2000006.魅力":3,//场上至少存在1张闲置的魅力大于等于3，且ID为2000006的卡牌
"table_have.激情<=":3,//场上所有闲置卡牌的激情总值小于等于3
//ps:因为激情标签在一张卡内不会叠加，所以可以用这种写法来表示场上带有激情标签的闲置卡牌小于等于3张。
"table_have.妻子.魅力":3,//场上至少存在1张闲置的魅力大于等于3，且拥有妻子标签的卡牌
"table_have.char":1,//场上至少存在1张卡牌类型为char的卡牌
"table_have.char.魅力":3,//场上至少存在1张闲置的魅力大于等于3，且类型为char的卡牌
--------------特殊写法------------------------------
"table_have.体魄>20.count":3,//场上闲置的卡牌至少存在3张大于20体魄的卡牌
```
- 标识不存在时候，在指令前面加【!】，这个算通用写法，在其他条件指令中也可以使用。
```json
-------------范例--------------------
"!table_have.2000063.激情": 1, //不存在闲置的，有激情标签的，且ID为2000063的卡牌
```
##### have——当前游戏存在对应卡牌(或不存在)
- 指令格式："have.筛选器":1（"!have.筛选器":1）
- 包含了闲置的和在仪式中的卡牌
- 配置格式和[table_have]一致，不做赘述了
##### hand_have——当前游戏在手牌存在对应卡牌(或不存在)
- 指令格式："hand_have.筛选器":1（"!hand_have.筛选器":1）
- 仅包含了手牌区的
- 配置格式和[table_have]一致，不做赘述了
##### rite——当前游戏存在对应的仪式（或不存在）
- 指令格式："rite":仪式id（"!rite":仪式id）
```json
-------------范例--------------------
"rite":5000001, //存在编号为5000001的仪式（5000001就是治理家业）
"!rite":5000001, //不存在编号为5000001的仪式（5000001就是治理家业）
```
##### counter——当前游戏存在对应数值的计数器
- 指令格式："counter.计数器id":1
- 计数器：设计者选定一个唯一编号，可以在命令中给这个编号赋予数值，也可以在条件中检定这个数值的大小，计数器在赋值前数值默认为0，每局游戏的计数器各自独立存在，游戏结束对应的计数器也会被删除。
- 如果要表示小于，等于，小于等于，大于等于，大于，可添加对应的符号
    - 如果不配置符号，默认是大于等于
```json
-------------范例--------------------
"counter.7000001":1, //编号为7000001的计数器数值大于等于1
"counter.7000001<":3, //编号为7000001的计数器数值小于3
```
##### global_counter——当前游戏存在对应数值的全局计数器
- 指令格式："global_counter.计数器id":1
- 全局计数器：游戏结束也不会被移除的计数器，影响全局，一般被用在成就或商店解锁条件。
##### 通用语法
- any：实现任意一个
- all：全部实现
- 不配置默认为all
```json
-------------在槽位中使用的范例--------------------
"s1":            
  {
      "condition": 
      {
	      "any":{ //主角或妻子
	    	 "is":2000001,
	   	     "is":2000006,
	      },
      	  "all":{ //属性必须是战斗大于等于5，且魅力大于等于4
      	     "战斗":5,
     	     "魅力":4,
	      },
      },
      "open_adsorb":0,
      "is_key":0,
      "is_empty":1,
      "is_enemy":0,
      "text":"主角或妻子"
  }
```
### 条件-仪式内
说明：仪式结算时激活对应结果的条件。
##### is——是什么id的卡牌
- 指令格式："is":卡牌id
- 注意：在槽位中使用和在结果实现的条件中使用是两个意思
    - 在槽位中使用：当前槽位放的是什么id的卡牌
    - 在结果实现的条件中使用：当前仪式中存在对应ID的卡牌
```json
-------------在槽位中使用的范例--------------------
"s1":            
  {
      "condition": 
      {
          "is":2000013, //2000013是流浪剑客，这个槽位只能放流浪剑客
      },
      "open_adsorb":0,
      "is_key":0,
      "is_empty":1,
      "is_enemy":0,
      "text":"流浪剑客"
  },
 -------------在结果实现的条件中使用--------------------
{
      "condition": {
           "is":2000013, //流浪剑客在仪式中
      },
      "result_title":"你看到了一个人影",
      "result_text": "是他！那个流浪剑客！",
      "result": 
      {
      },
      "action": {
      }
  }
```
##### type——是什么类型的卡牌
- 指令格式："type":"卡牌类型"
- 注意：在槽位中使用和在结果实现的条件中使用是两个意思
    - 在槽位中使用：当前槽位放的是什么类型卡牌
    - 在结果实现的条件中使用：当前仪式中存在对应类型的卡牌
```json
-------------在槽位中使用的范例--------------------
"s1":            
  {
      "condition": 
      {
          "type":"sudan", //这个槽位只能放类型为sudan的卡牌，苏丹卡都是这种类型的卡牌
      },
      "open_adsorb":0,
      "is_key":0,
      "is_empty":1,
      "is_enemy":0,
      "text":"苏丹卡"
  },
 -------------在结果实现的条件中使用--------------------
{
      "condition": {
           "type":："sudan", //这个仪式放了苏丹卡
      },
      "result_title":"测试文本",
      "result_text": "测试文本",
      "result": 
      {
      },
      "action": {
      }
  }
```
##### 卡牌tag——带有什么标签
- 指令格式："卡牌tag":1
- 注意：在槽位中使用和在仪式结果实现的条件中使用是两个意思
    - 在槽位中使用：当前槽位放的是拥有对应数值标签的卡牌
    - 在结果实现的条件中使用：当前仪式中拥有对应数值标签的友方卡牌
```json
-------------在槽位中使用的范例--------------------
"s1":            
  {
      "condition": 
      {
          "激情":1, //这张卡得有激情标签
          "魅力":3, //这张卡的魅力需要大于等于3
          "智慧<":5, //这张卡的智慧需要小于5
      },
      "open_adsorb":0,
      "is_key":0,
      "is_empty":1,
      "is_enemy":0,
      "text":"他是谁"
  },
 -------------在结果实现的条件中使用--------------------
{
      "condition": {
          "战斗":10, //任意槽位存在战斗大于等于10的卡牌       
      },
      "result_title":"测试文本",
      "result_text": "测试文本",
      "result": 
      {
      },
      "action": {
      }
  }
```
##### cost.卡牌tag——放置可以堆叠的卡牌
- 指令格式："cost.卡牌tag":1
- 一般仅在槽位中使用，被用做放置可以堆叠的卡牌槽位上的条件
```json
-------------在槽位中使用的范例--------------------
"s1":            
  {
      "condition": 
      {
          "cost.金币":5, //可以放5张金币（因为每张金币卡的金币标签是1点）
          "cost.金币":[1,5], //可以放最少1张，最多5张金币
      },
      "open_adsorb":0,
      "is_key":0,
      "is_empty":1,
      "is_enemy":0,
      "text":"金币"
  }
```
##### s编号——对应槽位编号存在对应卡牌（或不存在）
- 指令格式："s编号.筛选器":1
- 穷举筛选器配置格式
```json
"s编号":1,
"s编号.卡牌tag":1,
"s编号.type":"卡牌类型", 
-------------范例--------------------
{
    "condition": {
         "s1":1, //槽位s1非空
         "s2.激情":1, //槽位s2的卡牌有激情tag
         "s3.体魄>=":5, //槽位s3的卡牌体魄大于等于5
         "s4.type":"item", //槽位s4的卡牌类型需要是item
    },
    "result_title":"测试文本",
    "result_text": "测试文本",
    "result": 
    {
    },
    "action": {
    }
}
```
##### r编号:——检定对应编号的计算公式计算出的骰子投出的成功数
- 指令格式："r编号:公式":[成功数,5]
```json
-------------范例--------------------
{
    "condition": {
         "r1:s1.战斗+s2.体魄>=":[1,5], 
//s1卡牌的战斗加上s2卡牌的体魄获得的骰子数量，成功数必须大于等于1
         "r2:战斗+体魄>=":[3,5],
//友方所有战斗加上体魄获得的骰子数量，成功数必须大于等于3
         "r3:战斗+e(体魄)>=":[8,5],
//友方所有卡牌的战斗减去敌方所有卡牌的战斗获得的骰子数量，成功数必须大于等于8    
    },
    "result_title":"测试文本",
    "result_text": "测试文本",
    "result": 
    {
    },
    "action": {
    }
}
```
##### f:——检定对应属性的大小
- 指令格式："f:公式":1
```json
 -------------范例--------------------
{
    "condition": {
         "f:s1.战斗+s2.体魄>=":5, 
         //s1卡牌的战斗属性加上s2卡牌的体魄属性大于等于5
         "f:战斗+体魄>=":5,
         //友方的所有卡牌的战斗属性加上体魄属性大于等于5
         "f:战斗-e(战斗)>=":5,
         //友方所有卡牌的战斗属性减去敌方所有卡牌的战斗属性大于等于5    
    },
    "result_title":"测试文本",
    "result_text": "测试文本",
    "result": 
    {
    },
    "action": {
    }
}
```
### 条件-卡牌内
说明：卡牌在仪式结算时激活额外动作的条件。
##### is_rite——当前卡牌在什么id的仪式内
- 指令格式："is_rite":仪式id
```json
-------------在卡牌中使用的范例--------------------
"2000001": {
        "id": 2000001,
        "name": "阿尔图",
        "title": "被选中的人",
        "text": "你自己，一个卷入苏丹游戏的可悲之人。",
        "pops": [],
        "card_favour": "",
        "type": "char",
        "tips": "",
        "rare": 3,
        "resource": ["cards/2000001","cards/2000001_1","cards/2000001_2"],
        "tag": {
            "体魄": 3,
            "魅力": 2,
            "智慧": 1,
            "男性": 1,
            "贵族": 1,
            "主角": 1,
            "战斗": 2,
            "社交": 1,
            "已拥有": 1,
            "支持": 1
        },
        "card_vanishing": 0,
        "vanish": {
            "over": 13
        },
        "equips": [
            "武器",
            "服装",
            "饰品"
        ],
        "is_only": 1,
        "post_rite":[
            {
                "condition": {       
                    "is_rite":5000001, //当前卡牌正在参与的仪式是否是治理家业（5000001）                          
                },
                "result_text": "完成治理家业，社交+1",
                "result": 
                {
                    "self+社交":1,//仪式结算时，当前卡牌社交+1
                },
                "action": {
                }
            }
        ]
    }
```

```json
-------------在卡牌中使用的范例--------------------

        "post_rite":[

                },

                "result": 
                {
                    "self+魅力":1, //仪式结算时，当前卡牌魅力+1
                },
                "action": {
                }
            }
        ]
    }
```

##### parent/self.卡牌tag——带有什么标签
- 指令格式："parent.卡牌tag":1，指向装备了当前卡牌的卡牌
- 指令格式："self.卡牌tag":1，指向自己
```json
-------------在卡牌中使用的范例--------------------
.......
        "post_rite":[
            {
                "condition": {       
                    "parent.受伤":1, 
                    //当前卡牌正在参与的仪式中，装备了当前卡牌的卡牌是否存在受伤标签 
                    "self.装备":1,
                    //当前卡牌存在装备标签                      
                },
                "result_text": "梅姬，你也在啊",
                "result": 
                {
                    "clean.parent":1, //仪式结算时，装备了当前卡牌的卡牌会被移除
                },
                "action": {
                }
            }
        ]
    }
```
