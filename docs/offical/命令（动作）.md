# 命令（动作）
说明：幕后/仪式结算结果被激活时，可以执行的动作。获得卡牌，生成新的仪式都属于这类动作。
##### card——获得卡牌
- 指令格式："card":卡牌id
```json
-----------范例如下-------------------
"card":2000013,//生成2000013卡牌
--------------特殊写法------------------------------
"card":[2000013,"追随者+1"],//生成2000013卡牌，并且给2000013添加追随者标签
"金币":10,//特化的写法，直接获得10张金币卡
"card":[2000755,"数量+10"], //生成10张id为2000755的卡牌
"card":[2000032,"数量+10","重投+1"], //生成10张id为2000032的卡牌，并为这些卡牌添加重投
```
##### rite——生成仪式
- 指令格式："rite":仪式id
```json
"rite":5000013,//生成编号为5000013的仪式
```
##### copy——复制指定槽位的卡牌
- 说明：仅在仪式中有用
- 指令格式："copy.槽位编号":1
```json
"copy.s1":1,//复制一张槽位s1的卡牌
```
##### uprare——提升指定卡牌的品级1次
- 指令格式："指定卡牌.uprare":1
```json
"s1.uprare":1,//将当前仪式槽位s1的品级提升1级
"s1.uprare":-1,//将当前仪式槽位s1的品级降低1级
"table.2000001.uprare":1,//将当前闲置的，id为2000001的卡牌品级提升1级
"table.妻子.uprare":1,//将当前闲置的，且带有妻子标签的卡牌品级提升1级
"table.2000001.宰相.uprare":1,
//将当前闲置的，id为2000001且带有宰相标签的卡牌品级提升1级
"total.2000001.uprare":1,//将当前存在的，id为2000001的卡牌品级提升1级
"total.妻子.uprare":1,//将当前存在的，且带有妻子标签的卡牌品级提升1级
"total.2000001.宰相.uprare":1,
//将当前存在的，id为2000001且带有宰相标签的卡牌品级提升1级
```
##### event_on/off——激活或关闭指定幕后
- 指令格式：
    - 激活幕后："event_on":幕后id
    - 关闭幕后："event_off":幕后id
```json
"event_on":5300001,//激活5300001的幕后
"event_off":5300001,//关闭5300001的幕后
--------------特殊写法------------------------------
"event_off":1,//关闭所有已激活的幕后
```
##### clean——移除
- 指令格式：
    - 移除卡牌："clean.指定卡牌":数量
    - 移除仪式:   "clean.rite":仪式id
```json
-----------移除卡牌-------------------
"clean.s1":5,//将当前仪式s1槽位上的卡牌移除5张
"clean.2000001":1,//将当前仪式id为2000001的卡牌移除
"table.clean.妻子":1,//移除一张闲置的，带有妻子标签的卡牌
"table.clean.2000001.受伤":1,//移除一张闲置的，id为2000001且带有受伤标签的卡牌
"table.clean.item":0,//移除所有类型为item的卡牌
"table.clean.item|!2000545|!2000544":0,
//除了2000545和2000544之外，移除所有类型为item的卡牌
--------------特殊写法------------------------------
"clean.rite":5000001,//将当前id为5000001的仪式从场上移除
"clean.rite":1,//移除当前场上的所有仪式
```
##### loot——随机掉落
- 指令格式："loot":随机池id
```json
-----------范例如下-------------------
"loot":6000001,//激活随机池6000001掉落对应物品
--------------特殊写法------------------------------
"loot.已拥有+1":6000001,//激活随机池6000001掉落对应物品，并给掉落的物品添加已拥有标签
"loot":[6000071,6000073]，//优先掉落6000071中的物品，若6000071无东西可掉，则掉落6000073中的物品。
```
##### =/-/+卡牌tag——给指定卡牌设置，减少，增加对应标签
- 指令格式：
    - 设置："指定卡牌=卡牌tag":数值
    - 减少："指定卡牌-卡牌tag":数值
    - 增加："指定卡牌+卡牌tag":数值
- 穷举不同方式指定卡牌的写法
```json
-----------在当前仪式内指定卡牌-------------------
"s槽位编号=/-/+卡牌tag":数值, 
"卡牌id=/-/+卡牌tag":数值,
-----------在闲置的卡牌中指定卡牌-------------------
"table.卡牌id=/-/+卡牌tag":数值,
"table.卡牌tag=/-/+卡牌tag":数值,
"table.类型=/-/+卡牌tag":数值,
-----------在存在的卡牌中指定卡牌（仪式内/闲置）-------------------
"total.卡牌id=/-/+卡牌tag":数值,
"total.卡牌tag=/-/+卡牌tag":数值,
"total.类型=/-/+卡牌tag":数值,
-----------范例如下-------------------
"s1=体魄":0,//将当前仪式s1槽位的卡牌体魄设置为0
"s1-体魄":1,//将当前仪式s1槽位的卡牌体魄减1
"s1+体魄":3,//给当前仪式s1槽位的卡牌体魄加3
"2000001=体魄":0,//将当前仪式id为2000001的卡牌体魄设置为0
"2000001-体魄":1,//将当前仪式id为2000001的卡牌体魄减1
"2000001+体魄":3,//给当前仪式id为2000001的卡牌体魄加3
"table.2000001=体魄":0,//将当前处在闲置状态，且id为2000001的卡牌体魄设置为0
"table.妻子-体魄":1,//将当前处在闲置状态，且带有妻子标签的卡牌体魄减1
"table.char+体魄":3,//将当前处在闲置状态，且类型为char的卡牌体魄加3
"total.2000001=体魄":0,//将当前存在且id为2000001的卡牌体魄设置为0
"total.妻子-体魄":1,//将当前存在且带有妻子标签的卡牌体魄减1
"total.char+体魄":3,//将当前存在且类型为char的卡牌体魄加3
```
##### =/-/+counter——给指定局内计数器设置，减少，增加对应数值
- 说明：
    - 计数器作用：比如玩家完成事件A的时候设置了计数器7000001，事件B可以通过检定计数器7000001来判断事件A是否已经触发。
    - 局内计数器，仅在本局游戏内有效。
    - 计数器的id为7开头的7位数字，需要时摘取未使用的数字即可，比如7000001
    - 计数器数值为正整数，最小值为0，默认为0。
- 专属计数器：
    - 苏丹的游戏内通过增加特定编号的计数器来变更玩家的声望，金骰子以及回到上一回合次数
    - 善名：7100001
    - 恶名：7100002
    - 权势：7100003
    - 侠名：7100004
    - 灵视：7100005
    - 金骰子数：7100006
    - 回到上一回合次数：7100007
- 指令格式：
    - 设置："counter=计数器id":数值
    - 减少："counter-计数器id":数值
    - 增加："counter+计数器id":数值
```json
"counter=7000001":1,//将7000001计数器数值设置为1
"counter-7000001":1,//将7000001计数器数值减1
"counter+7000001":3,//将7000001计数器数值加3
```
##### =/-/+global_counter——给指定全局计数器设置，减少，增加对应数值
- 说明：
    - 全局计数器，全局有效，一般用在一千零一夜和命运商店的商品解锁中。
    - 计数器的id为7开头的7位数字，需要时摘取未使用的数字即可，比如7200001
    - 计数器数值为正整数，最小值为0，默认为0。
- 指令格式：
    - 设置："global_counter=计数器id":数值，设置后立即生效
    - 减少："global_counter-计数器id":数值，本局游戏结束时生效
    - 增加："global_counter+计数器id":数值，本局游戏结束时生效
##### +/-/~equip——给指定卡牌装备，移除，卸载对应卡牌
- 说明：通过指令实现A卡装备到B卡上的效果，不需要卡牌A是装备卡也能实现装备效果，就是装备了没法自主脱下来了。游戏内的受伤效果就是通过装备受伤卡来实现的。
- 指令格式：
    - 装备："指定卡牌+equip":卡牌id
    - 移除："指定卡牌-equip":筛选器
    - 卸载："指定卡牌~equip":筛选器
        - 被卸载的卡牌还会留在场上
```json
指定卡牌：s槽位编号，卡牌id，table，total
筛选器:卡牌id，卡牌tag
-----------范例如下-------------------
"s1+equip":2000013,//创建一张2000013并将其装备到当前仪式s1槽位上的卡牌上
"2000001-equip":"武器",//将当前仪式id为2000001的卡牌身上带有武器tag的的卡牌移除
"table.char~equip":"服装",//将当前闲置的类型为char的卡牌身上带有服装tag的的卡牌全部卸载
"total.2000001-equip":2000013,//将当前场上id为2000001卡牌身上装备的2000013移除
-----------特殊写法-------------------
"s1+s2":1,//将当前仪式的s2上的卡牌装备到s1槽位上的卡牌上
```
##### +/-/equip_slot——给指定卡牌添加或移除对应装备槽位
- 说明：通过指令实现增减卡牌身上的装备槽位
- 指令格式：
    - 添加："指定卡牌+equip_slot":装备槽位代码
    - 移除："指定卡牌-equip_slot":装备槽位代码
```json
装备槽位代码：
饰品：accessory
武器：weapon
服装：cloth
驯兽：animal_handling
-----------范例如下-------------------
"s2+equip_slot":"animal_handling",  //获得动物槽位
```

##### pop——生成气泡
![图片](https://docimg6.docs.qq.com/image/AgAABS6GRkcrqkZPN2lPu6OR_qnIDIpn.png?w=393&h=339)
- 指令格式：
    - 给当前仪式内指定卡牌添加气泡："pop.编号.指定卡牌":"文本"
    - 给当前手牌指定卡牌添加气泡："hand_pop.编号.指定卡牌":"文本"
    - 给指定仪式添加气泡："rite_pop.编号.指定卡牌":"文本"
```json
-----------范例如下-------------------
"pop.5000001_01.s1":"测试文本"，
//当前仪式槽位s1上的卡牌显示上"测试文本"气泡
"pop.5000001_01.妻子":"测试文本"，
//当前仪式带有妻子标签的卡牌上显示"测试文本"气泡
"pop.5000001_01.2000001":"测试文本"，
//当前仪式id为2000001的卡牌上显示"测试文本"气泡
"rite_pop.5310107_01.5005101":"测试文本", 
//当前存在的id为5005101的仪式图标上显示"测试文本"气泡
"hand_pop.5310107_02.2000001":"测试文本",
//当前玩家可见的手牌区id为2000001的卡牌上显示"测试文本"气泡
```

##### 语法
- 随机选择一个动作执行：
    - 指令格式："choose":{}
    ```json
---------------------范例----------------------
{
      "condition": {
      },
      "result_title":"",
      "result_text": "",
      "result": 
      {
          "choose":{ //从A，B，C这3个动作中随机一个执行
              "counter+7100006":1,  //动作A
              "card":2000986,   //动作B
              "card":2000724  //动作C
          },
      },
      "action": {
      }
 }
```
- 延迟X回合执行指定动作：
    - 指令格式："delay":{}
    ```json
---------------------范例----------------------
{
      "condition": {
      },
      "result_title":"",
      "result_text": "",
      "result": 
      {
          "delay":{ //延迟3回合执行动作A
		              "id":50008631, //延迟编号，不可使用已有的编号
		              "round": 3, //延迟3回合执行
		              "rite":5000863  //动作A
                  },
      },
      "action": {
      }
 }
```
- 让执行的动作玩家不可见：
    - 指令格式：
        - 在幕后（event）中执行的动作不可见："no_prompt":{}
        - 在仪式（rite）中执行的动作不可见："no_show":{}
    ```json
---------------------幕后中执行的范例----------------------
 "no_prompt":{ //玩家将无法在弹框中看到2000772生成的表现
                  "card":2000772
              },
              "prompt":{ //弹框
                  "id":"5300276_prompt_1",
                  "text":"弹框文本",
                  "icon":"cards/2000064"
              },
---------------------仪式中执行范例----------------------
{
      "condition": {
      },
      "result_title":"",
      "result_text": "",
      "result": 
      {
          "no_show":{ //在结算时玩家将无法看到卡牌2000013生成的表现
		              "card":2000013
                    },
      },
      "action": {
      }
 }

```

##### 改变已有卡牌，仪式的名字和描述
- 指令说明
    - change_card_name.key.filter 变更filter对应卡牌名称, key为这个指令的唯一编号，方便程序提取做国际化处理
    - change_card_text.key.filter 变更filter对应卡牌描述, key为这个指令的唯一编号，方便程序提取做国际化处理
    - change_rite_name.key.riteId 变更riteId对应的仪式名称(包括后续仪式), key为这个指令的唯一编号，方便程序提取做国际化处理
```json
//----------------范例---------------------
"change_card_name.test_01.2000001":"阿尔图一号" //将2000001这张卡改名为阿尔图一号
"change_card_text.test_02.2000001":"这是阿尔图的复制人" //将2000001这张卡的描述改为这是阿尔图的复制人
"change_rite_name.test_03.5000001":"花天酒地" //将5000001这这个仪式的名字改为花天酒地
```
##### 调用改名的弹框
- 指令说明
    - change_name
```json
//----------------范例---------------------
"change_name":0 //调用改名弹框，指向带有主角tag的卡牌，命名框内的默认名称为对应卡牌的名字
"change_name":卡牌id //调用改名弹框，指向对应id的卡牌，命名框内的默认名称为对应卡牌的名字
```
