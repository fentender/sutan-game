# 触发时机
说明：幕后（event）触发条件检定的可用时机（timing）
##### 每过n回合开始的时候
- 指令格式："round_begin_ba":n
```json
"round_begin_ba":1, //每回合开始时
"round_begin_ba":2, //每过2回合开始的时候，依此类推）
"round_begin_ba":[2,5], //每次2~5随机一个整数，包括2和5
-----------范例如下-------------------
{
        "id": 5300005,
        "text": "法务官之死",     
        "is_replay": 1, //是否可以重复触发
        "auto_start": false,
        "start_trigger": false, // 是否启动就立即开始校验条件
        "on":{   //时机
            "round_begin_ba": 1 //每回合开始时
        },
        "condition":{  //条件
            "table_have.2000164":1  //桌面有法官空闲
        },
        "settlement": [    //效果
            {
                "tips_resource":"", 
                "tips_text":"",
                "action":{
                    "rite":5000505,
                    "success":{
                        "event_off":5300005
                    }
                }
            }
        ]
    }
```
##### 回到上一回合时
![图片](https://docimg6.docs.qq.com/image/AgAABS6GRkdtLSxNOrdEUblkC-HdEHX8.png?w=289&h=203)
- 指令格式："back_to_prev_round_end":1
##### 生成卡牌的时候
- 指令格式："card_born":卡牌id
```json
"card_born":2000013, //生成卡牌2000013时
"card_born":[2000013,2000014], //生成卡牌2000013或2000014时
```
##### 卡牌被移除时
- 指令格式："card_clean":卡牌id或"card_clean.卡牌tag":1
```json
"card_clean":2000013, //卡牌2000013被移除时
"card_clean":[2000013,2000014], //卡牌2000013或2000014被移除时
"card_clean.激情":1, //带有激情标签的卡牌被移除时
```
##### 卡牌生命周期结束时
说明：只有配置了生命周期的卡牌才会触发，比如7回合生命周期的苏丹卡。生命周期结束的时候还在仪式中，需要等仪式结束后才算。
- 指令格式："card_dead":卡牌id
```json
"card_dead":2010001, //卡牌2010001生命周期结束的时候
```
##### 仪式（rite）手动启动时
![图片](https://docimg8.docs.qq.com/image/AgAABS6GRkchS0moJYtGNLlfjrh86cGT.png?w=538&h=681)
- 指令格式："rite_start":仪式id
```json
"rite_start":5000001, //启动5000001时
"rite_start":[5000001,5000002], //启动5000001或5000002时
```
##### 仪式（rite）启动时
说明：包含了设置为自动启动的仪式和手动启动的仪式
- 指令格式："rite_begin":仪式id
```json
"rite_begin":5000001, //启动5000001时
"rite_begin":[5000001,5000002], //启动5000001或5000002时
```
##### 仪式（rite）非正常关闭时
说明：特指仪式等待时间结束后关闭，或通过幕后动作移除时
- 指令格式："rite_clean":仪式id
```json
"rite_clean":5000001, //5000001非正常关闭
"rite_clean":[5000001,5000002], //5000001或5000002非正常关闭时
```
##### 仪式（rite）正常关闭时
说明：特指仪式结算后，对应的结算界面关闭时
- 指令格式："rite_end":仪式id
```json
"rite_end":5000001, //5000001正常关闭时
"rite_end":[5000001,5000002], //5000001或5000002正常关闭时
--------------特殊写法------------------------------
"rite_end":1, //任意仪式正常关闭时
```
##### 打开对应仪式（rite）界面时
说明：玩家点击仪式图标，打开仪式界面时
- 指令格式："open_rite":仪式id
```json
"open_rite":5000001, //5000001打开时
"open_rite":[5000001,5000002], //5000001或5000002打开时
```
##### 指定局内计数器（counter）变动时
- 指令格式："counter":计数器id
```json
"counter":7000001, //7000001数值发生变化时，增减都算
"counter":[7000001,7000002], //7000001或7000002发生变化时
--------------特殊写法------------------------------
"counter":1, //任意计数器发生变化时
```
##### 指定全局计数器（global_counter）被设置时
说明：只有"global_counter=7220001":1这种写法才算设置，"global_counter+7220001":1或"global_counter-7220001":1不算设置。
- 指令格式："global_counter":计数器id
```json
"global_counter":7220001, //7220001被设置时
--------------特殊写法------------------------------
"global_counter":1, //任意全局计数器被设置时
```
