- 如何配置坐标
    - 配置固定坐标
        - 如果两个配置了一样编号固定坐标的仪式同时生成，那么它们就会重叠在一起
    ```json
"id": 5000001,
        "name": "治理家业",
        "text": "人们仍然愿意来到你的屋檐下，分享言语、允诺，与机会。",       
        "tips": "",
        "mapping_id":8001002,
        "once_new":1,
        "round_number": 1,
        "waiting_round":0,
        "waiting_round_end_action":[
        ],
        "method_settlement": "",
        "auto_begin": 1,
        "auto_result":1,
        "location": "自宅:1", //只在自宅的编号1坐标上生成
```
    - 配置随机坐标
        - 配置了随机坐标的两个仪式即使仪式生成时编号一样也会自适应的推开合适的距离
```json
"id": 5000016,
        "name": "夜间的战斗",
        "text": "阅读可以提升自己，养成阅读的习惯吧",       
        "tips": "",
        "mapping_id":8001005,
        "once_new":0,
        "round_number": 1,
        "waiting_round":1,
        "waiting_round_end_action":[
        ],
        "method_settlement": "",
        "auto_begin": 0,
        "auto_result":0,
        "location": "自宅:[2,12]", //在自宅的2~12编号的坐标中随机生成
```
##### 因为仪式槽位数量，导致仪式无法成功生成的问题
- 仪式的槽位不能超过15个，推荐10个以内，超出10个在默认的仪式背景里会有超框等显示问题
- 如果用户修改增加现有仪式的槽位数量，建议删除仪式的mapping_id参数（如果有配置的话），否则就有可能导致槽位数量和背景的预设的槽位数量对不上而导致仪式无法生成的问题。
```json
"id": 5006531,
        "name": "复仇的滋味",
        "text": "这本不知被谁放在你卧室门口的书，记录着怎样的故事呢？",       
        "tips": "",
        "mapping_id":8001005, //如果增加槽位数量，建议把这一条参数删除
        "once_new":1,
        "round_number": 1,
```
- "mapping_id"索引的是当前仪式的背景模板，包含了仪式的背景，槽位数量，槽位位置信息。如果仪式没有配置"mapping_id"则默认使用基础模板背景。基础模板推荐使用不多于10个槽位，最大可支持15个槽位。
    - 目前版本暂不支持仪式背景的预览和编辑
##### have条件使用错误，找不到对应的卡牌导致仪式或幕后生成失败
- 卡牌在游戏内存在分为几种状态
    - [不存在]：未生成，或者已经被移除的卡牌属于这一类
        - 这类卡牌无法被任何条件或动作指定
        - 特殊：被装备到其他卡牌身上的卡牌无法被have找到，可以灵活运用可继承的tag来查找
    - [存在，在手牌]：已存在或生成的卡牌，目前在玩家的手牌区（背包）里
        - 这类卡牌可以被have，table_have，hand_have这类条件找到
        - 这类卡牌可以被table，total这类动作指定
    - [存在，不在手牌区，闲置]：已存在或生成的卡牌，存在在玩家不可见的“牌池”里
        - 这类卡牌可以被have，table_have这类条件找到
        - 这类卡牌可以被table，total这类动作指定
    - [存在，不在手牌区，在仪式内]：已存在或生成的卡牌，正在仪式的槽位中
        - 这类卡牌可以被have这类条件找到
        - 这类卡牌可以被total这类动作指定
##### 用event生成仪式失败
- 常见的原因：生成的仪式被设置成必须要自动吸取指定卡牌，而对应的卡牌不存在或者未处于闲置状态
- 解决办法：在condition中添加检定对应卡牌是否闲置的条件
```json
-------------------范例---------------------------------
{
        "id": 5300002,
        "text": "向神殿求助",  
        "is_replay": 0, 
        "auto_start": false, 
        "start_trigger": true,
        "on":{
            "round_begin_ba": 1 //每回合开始时
        },
        "condition":{
            "table_have.2000021":1  //桌面有祭司空闲
        },
        "settlement": [    //效果
            {
                "tips_resource":"", 
                "tips_text":"",
                "action":{
                    "rite":5000508
                }
            }

        ]
    }
```
- 用event生成的卡牌没有出现在手上
    - 检查生成的卡牌身上是否有【已拥有】或【追随者】的标签
    - 如果没有的话，在生成的时候要一起加上
```json
-------------------范例---------------------------------
{
        "id": 5300002,
        "text": "向神殿求助",  
        "is_replay": 0, 
        "auto_start": false, 
        "start_trigger": true,
        "on":{
            "round_begin_ba": 1 //每回合开始时
        },
        "condition":{
        },
        "settlement": [    //效果
            {
                "tips_resource":"", 
                "tips_text":"",
                "action":{
                    "card":[2000771,"已拥有+1"]
                }
            }

        ]
    }
```
##### 手上有一张卡牌，明明满足槽位条件，但无法放入仪式槽位里
- 常见的原因：这张卡牌带有“吸附指定”效果标签
- 解决办法：如果这个标签在设定之外，就将它删除掉，或者根据需要修改吸附指定的配置
##### 因为仪式设置了必须吸指定卡牌，但对应卡牌不存在导致仪式生成失败
- 常见的原因：仪式自动吸取的必要卡牌未处在闲置状态
- 解决办法：在生成这个仪式的幕后里添加这张卡必须闲置的条件
