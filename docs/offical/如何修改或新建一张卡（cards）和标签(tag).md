# 如何修改或新建一张卡（cards）和标签(tag)
### 卡牌（cards）说明
- 游戏内的角色，道具，特殊的状态等都可以卡牌的形式表现
- 卡牌是标签（tag）的载体
### 如何新建一张卡牌（cards）
- 所有卡牌都配置在cards.json表中
- 在cards表中新建未被使用的数字2开头的7位数作为id的卡牌
### cards表的数据结构说明
```json
{
    "2000001": { //未被使用的数字2开头的7位数id
        "id": 2000001,//和上面一致
        "name": "阿尔图", //卡牌名称
        "title": "被选中的人", //卡牌称号
        "text": "你自己，一个卷入苏丹游戏的可悲之人。", //卡牌描述
        "pops": [],//参数要保留，但没有用，不用配置
        "card_favour": "",//参数要保留，但没有用，不用配置
        "type": "char", //卡牌类型，目前可用的有char，sudan，item这3类
        "tips": "",//参数要保留，但没有用，不用配置
        "rare": 3,//卡牌的品级，1~4分别对应岩石，青铜，白银和黄金
        "resource": ["cards/2000001","cards/2000001_1","cards/2000001_2"],
        //当前卡牌可用的卡面，默认显示第一个卡面
        //可以通过给卡牌设置"指定当前卡牌=图片":0，切换成第一张卡面
        //"指定当前卡牌=图片":1，切换成第二张卡面
        //"指定当前卡牌=图片":2，切换成第三张卡面
        "tag": { //当前卡牌默认就持有的标签（tag）
            "体魄": 3, //对应的格式："标签名": 数值
            "魅力": 2,
            "智慧": 1,
            "男性": 1,
            "贵族": 1,
            "主角": 1,
            "战斗": 2,
            "社交": 1,
            "已拥有": 1,
            "支持": 1
        },
        "card_vanishing": 0, 
        //卡牌的生命周期，生命周期结束卡牌就会被移除
        "vanish": { //亡语，当前卡牌被移除时执行的动作
            "over": 13 //执行对应编号的结局
        },
        "equips": [ 
        //当前卡牌的装备槽位可以用的装备槽位有：武器，服装，饰品，驯兽，可配置多个
            "武器",
            "服装",
            "饰品"
        ],
        "is_only": 1 //标识当前卡牌是否是唯一，
    },
        "post_rite":[ //当前卡牌在仪式使用时额外执行的动作，设计者根据需要添加配置
            {
                "condition": {      
                },
                "result_text": "阿尔图驾到",
                "result": 
                {
                    "self+体魄": 1//在任意仪式使用后体魄+1
                },
                "action": {
                }
            }
        ]
}
```
### 标签（tag）说明
- 绑定在卡牌上，用于检定或带有特殊功能
- tag的中文名称一般会直接被用来检定，所以已有的tag中文名不建议做修改
### 如何新建一张标签（tag）
- 所有卡牌都配置在tag.json表中
- 在tag表中新建未被使用的数字3开头的7位数作为id的卡牌
### tag表的数据结构说明
```json
{
	"physique": { //tag的英文名称，不可重复
        "id": 3000001, //未被使用的数字3开头的7位数作为id的卡牌
        "name": "体魄", //tag的名称，直接使用中文
        "code": "physique", //tag的英文名称，同上
        "type": "buff", 
        //目前分为buff和attribute两类
        //一般可堆叠的属性用buff，标识状态有无的用attribute
        "text": "决定了角色的力量，敏捷与耐力，象征了身体的综合素质，也是形体健美的象征。",
        //表述文本
        "tips": "",//策划自己看的备注
        "resource": "tag_1",
        //tag的图标，配置对应图标的资源名。对应资源的放置在mod要求的对应文件地址中
        "can_add": 1,
        //设置当前tag是否可以堆叠，1为可堆叠，0为不可堆叠。不可堆叠无论怎么加都是1
        "can_visible": 1,
        //设置玩家是否可见，1为可见，0为不可见。
        "can_inherit": 1,
        //设置当前标签是否可以通过装备继承到被装备的卡牌上，1为可以，0为不可以
        //策划就是通过这个参数实现，将武器卡牌上的属性加到装备者的身上的
        "can_nagative_and_zero": 1,
        //设置当前tag的负值是否有意义，1为有意义，0为无意义
        //设置为1时，tag的值就支持负数，否则最小值就是0
        "fail_tag": [],,//参数要保留，但没有用，不用配置
        "tag_vanishing": 0,//参数要保留，但没有用，不用配置
        "tag_sfx": "",
        //用来设置带有了这个tag的卡牌，卡面会添加上对应的特效
        //暂不开放编辑
        "tag_rank": 99,//tag在卡牌上的显示顺序，数字越大越靠前
        "attributes": {}
    },
    "lock_152": {
        "id": 3020261,
        "name": "被驯服的人",
        "code": "lock_152",
        "type": "attribute",
        "text": "被马衔驯服的人，无法进行任何事",
        "tips": "",
        "resource": "tag_0",
        "can_add": 0,
        "can_visible": 1,
        "can_inherit": 0,
        "can_nagative_and_zero": 0,
        "fail_tag": [],
        "tag_vanishing": 0,
        "tag_sfx": "hurt",
        "tag_rank": 10,
        "attributes": {
            "吸附指定": 1 
   //配置了吸附指定，带有这个tag的卡牌，只能放入要求这个tag的仪式槽位中，无法被其他仪式槽位使用
        }
    },
}
```
### 带有专属功能的标签（tag）
- 已拥有/追随者：带有这个标签的卡牌会成为玩家可使用的手牌
- 装备：带有这个标签的卡牌才会被定义为“装备”，并配置装备槽位类型相关的标签才能实现装备穿脱
    - 比如一把匕首卡牌，它需要同时拥有装备，武器这两个tag，它才能自主穿脱到角色身上。
- 武器/服装/饰品/驯兽：对应卡牌装备槽位类型的标签，配合“装备”tag，可以让卡牌装备到有对应装备槽位的卡牌上
- 纵欲/奢靡/杀戮/征服：对应苏丹卡的专属标签
- 金币：金币卡的专属标签
- 消耗品：带有这个标签的卡牌，在任意仪式出场，如果没有额外处理默认仪式结算的时候就会被移除。
- 回收：在仪式中给指定槽位的消耗品添加回收标签，这张消耗品卡牌就不会被移除。
```json
"result": 
        {
            "s1+回收":1
        },
```
