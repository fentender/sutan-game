# 如何修改或新建一个幕后（event）
### 幕后（event）说明
- 顾名思义，在幕后根据玩家的行动产生动作反馈，可以理解为没有操作界面的仪式
- 常被用做生成弹框，选项弹框，仪式，卡牌等
### 如何新建一个幕后（event）
- 在mod提供的地址中新建未被使用的数字53开头的7位数id的json文件
    - 补充说明：53开头的7位数id是官方推荐的id段，制作者也可以使用任意开头未被使用的7位数
    只要不和已有的id重复就行。
![图片](https://docimg4.docs.qq.com/image/AgAABS6GRkcoYDUm8PpE7JM0ENeFTLHN.png?w=689&h=326)
### event的数据结构说明
```json
--------------以下为一个完整的event表结构-----------------------
{
        "id": 5300002, //未被使用的数字53开头的7位数id
        "text": "向神殿求助", //策划自己看的备注      
        "is_replay": 0, //是否可以重复触发，动作未执行都不算触发
        "auto_start": false, 
        //游戏初始化时（新游戏），用来设置是否默认激活。当前false为不激活
        //如果要在新手阶段（苏丹玩苏丹卡的时候）设置默认激活，改为"auto_start_init": [0],
        //如果要在正式阶段（主角玩苏丹卡的时候）设置默认激活，改为"auto_start_init": [1],        
        "start_trigger": true,
         // 当前event激活时，是否跳过时机，立即开始检定条件执行动作。当前true为是的
         // 改为false则代表不跳过时机。
        "on":{   //event激活后，在满足触发时机时来检定条件执行动作
            "round_begin_ba": 1 
            //根据策划需求选用可用的触发时机
            //如果配置多个触发时机，满足任意一个时机即可开始检定条件
        },
        "condition":{  //在满足触发时，开始检定条件
            "table_have.2000021":1  
            //根据策划需求选用可用的条件
            //根据策划需求可以配置多个，并灵活使用any，all等语法
            //如果配置为空则视作满足条件
        },
        "settlement": [ //条件检定通过开始执行具体的动作
            {
                "tips_resource":"", //参数要保留，但没有用，不用配置
                "tips_text":"", //参数要保留，但没有用，不用配置
                "action":{  //执行具体的动作
                    "rite":5000508
                    //根据策划需求选用可用的动作
                    //根据策划需求可以配置多个，并灵活使用choose，delay等语法
                }
            }

        ]
    }
```
### 如何在开场游戏（苏丹玩苏丹卡的阶段）和正式游戏（主角玩苏丹卡的阶段）配置初始幕后
- 在开场有配置初始幕后
    - 将"auto_start_init"配置为[0]
```json
---------------------范例---------------------------------------
{
        "id": 5310100,
        "text": "苏丹引导-day1-弹框1",
        "is_replay": 0, 
        "auto_start_init": [0], //在开场游戏中默认激活的幕后
        "start_trigger": false, 
        "on":{   
            "round_begin_ba": 1
        },
        "condition":{  
        },
        "settlement": [    
        {
            "tips_resource":"", 
            "tips_text":"",
            "action":{
                "prompt":{
                    "id":"5310100_prompt_01",
                    "text": "<size=172%>很</size>久很久以前，在一个遥远的国度里，有一位强大的苏丹……\n这位苏丹在八年前踩着自己父亲的尸体登上了王位，继承了这个巨大的王国。",
                    "icon": ["common/introduction","",""]
                },
                "event_on":5310101
            }
        }
        ]
    }
```
- 在正式游戏配置初始幕后
    - 将"auto_start_init"配置为[1]
    ```json
{
    "id": 5300097,
    "text": "第3天生成妓院",     
    "is_replay": 0, 
    "auto_start_init": [1],//在正式游戏中默认激活的幕后
    "start_trigger": false, 
    "on":{
        "round_begin_ba":1 //每回合刷一次
    },
    "condition":{
        "counter.7000060>=":3
    },
    "settlement": [    //效果 
        {
            "tips_resource":"", 
            "tips_text":"",
            "action":{
                "event_on":5300012, //激活欢愉之光
                "success":{
                    "prompt":{
                        "id": "5300097_prompt_01",
                        "text":"传单：我们这里有最好的姑娘，欢迎来玩啊！----欢愉之馆",
                        "icon":"cards/2000792"
                    }
                }
            }
        }

    ]
}
```
### 范例1：用event生成一个rite（仪式）
```json
范例说明：当主角善名10的时候生成仪式5000011，只触发1次
--------------以下为一个完整的event表结构-----------------------
{
        "id": 5300001, //未被使用的数字53开头的7位数id
        "text": "向神殿求助", //策划自己看的备注      
        "is_replay": 0, //是否可以重复触发
        "auto_start_init": [1], //默认激活        
        "start_trigger": false, //不用立即检定条件
        "on":{
            "round_begin_ba": 1 //激活后在每回合开始的时候检定条件
        },
        "condition":{
            "counter.7100001":10 //检定善名大于等于10
        },
        "settlement": [ //条件检定通过开始执行具体的动作
            {
                "tips_resource":"", 
                "tips_text":"", 
                "action":{
                    "rite":5000011 //生成仪式
                }
            }
        ]
    }
```
### 范例2：用event生成一个prompt（弹框）
```json
范例说明：当主角善名10的时候生成弹框，每3~5回合触发1次，可重复触发
--------------以下为一个完整的event表结构-----------------------
{
        "id": 5300002,
        "text": "向神殿求助", //策划自己看的备注      
        "is_replay": 1, //重复触发
        "auto_start_init": [1], //默认激活        
        "start_trigger": false, //不用立即检定条件
        "on":{
            "round_begin_ba": [3,5] //每3~5回合的回合开始时
        },
        "condition":{
            "counter.7100001":10 //检定善名大于等于10
        },
        "settlement": [ //条件检定通过开始执行具体的动作
            {
                "tips_resource":"", 
                "tips_text":"", 
                "action":{
                    "prompt":{
                        "id":"5300002_prompt_01", 
                        //弹框的编号，没有严格格式，只要不重复方便检索就好
                        "text":"你是好人", //弹框文本，支持富文本
                        "icon":["cards/2000123","cards/2000001"]
                        //根据策划需求配置对应的弹框背景图，非必须
                        //最多可配置3张图，相对关系时左中右
                    }
                }
            }
        ]
    }
```
### 范例3：用event生成一个option（选项弹框）
```json
范例说明：当主角善名小于10的时候生成选项弹框，每3~5回合触发1次，可重复触发
--------------以下为一个完整的event表结构-----------------------
{
        "id": 5300003,
        "text": "向神殿求助", //策划自己看的备注      
        "is_replay": 1, //重复触发
        "auto_start": false, //默认不激活        
        "start_trigger": false, //不用立即检定条件
        "on":{
            "round_begin_ba": [3,5] //每3~5回合的回合开始时
        },
        "condition":{
            "counter.7100001<":10 //检定善名小于10
        },
        "settlement": [ //条件检定通过开始执行具体的动作
         {
            "tips_resource":"", 
            "tips_text":"",
            "action":{
                "option":{
                    "id":"5300003_option_01", //选项弹框的编号
                    "text":"你在路边遇到了一位向你求助的女士", //选项弹框的文本
                    "icon":"cards/2000123", //选项弹框的背景
                    "items": [
                        {
                            "text": "无视她", //选项1按钮上的文本
                            "tag": "op1"    //选项1的标记
                        },
                        {
                            "text": "帮助她",//选项2按钮上的文本
                            "tag": "op2"   //选项2的标记
                        }
                    ]
                },
                "case:op1": {  //选中选项1执行的动作
                    //根据策划需求选用可用的动作
                    //根据策划需求可以配置多个，并灵活使用choose，delay等语法
                    "prompt":{
                        "id":"5320036_prompt_01",
                        "text":"什么也没发生",
                    }
                },
                "case:op2": {
                    "counter+7100004": 1, //侠名+1
                    "prompt":{
                        "id":"5320036_prompt_02",
                        "text":"她向你表达了感谢",
                    }
                }
            }
        ]
    }
```
