# 变更日志：v1.1.2 → v1.2.0

> 时间范围：2026-04-13 至 2026-04-19

---

## 功能更新

冲突面板现在有

### 1. 更新数组匹配算法和合并机制

修复了工具合并时对于数组对应关系和顺序关系处理错误导致的bug。（但是某些mod的数组元素和本体差异太大，还是无法完全解决问题，某些mod可能需要人工校对。）

数组匹配算法：
对于例如settlement的数组，为了后续的合并操作需要计算mod和游戏本体中每个元素的对应关系，之前是仅使用guid或者根据数组索引进行匹配。现在引入了新的数组匹配算法，能够覆盖更多场景。（但是某些mod的修改还是无法准确计算对应关系，这个时候，可能需要人工校对。）

数组合并机制：
游戏中的settlement和settlement_prior的数组元素之间相对顺序是会影响到游戏机制，但是之前的时候没有考虑这点，导致某些mod合并后出现bug。

### 2. 新增三种合并可选模式

随着游戏大版本更新，现在创意工坊还存在大量的mod是基于旧版本开发，目前还没有进行更新。因此其中的过时mod直接应用会导致游戏本体更新的新增内容可能被覆盖。因此现在提供了合并模式：
(1). 正常合并。严格按照mod的修改对游戏本体文件进行覆盖，这种情况下旧版本mod可能会将游戏本体更新的内容删除，导致缺失游戏新版本内容。  
(2). 智能合并。选择性的应用mod对游戏本体的修改。保证旧版本mod不会删除游戏本体的新版本内容。但是这也可能会引入一些冲突，现在也没法保证绝对正确。
(3). 直接替换。直接将mod所涉及的文件直接整个替换，能够保证mod对游戏的修改严格实现。当你需要某个mod的功能必须实现时可以将其设为该模式，并且放置于优先级最高位置。

三种合并模式可以在全局和每个mod单独设置。但是所有合并模式都无法保证绝对无冲突，具体情况还需要自己检测合并面板是否存在冲突。

### 4. 性能优化（2026-04-18）

优化 `src/core/diff_formatter.py` 的差异格式化逻辑和 `src/gui/diff_dialog.py` 的渲染性能。改进了 Diff 对话框在大量差异数据时的加载速度，减少不必要的重复计算。`src/core/types.py` 新增缓存相关的类型支持。

### 7. 新增 Mod 和游戏版本检测功能（2026-04-19）

新增 `src/core/steam_time.py` 模块（+103 行），通过 Steam Workshop 的文件时间戳检测 Mod 的更新时间，并与游戏本体版本进行比对。`src/gui/mod_list.py` 新增版本状态的可视化展示，帮助用户判断哪些 Mod 可能因游戏更新而需要重新合并。`src/gui/json_editor.py` 也增加了相关信息的展示入口。
